Améliore toutes les pages média (films, séries, collections) pour qu’elles soient ultra complètes, fluides, accessibles et sans bug, avec une présentation du titre indépendante de l’affiche, une typographie immersive, et une structure visuelle cohérente.

1. Présentation du titre
   - Le titre du média doit être affiché en haut de la fiche, **indépendamment de l’affiche**.
   - Utiliser une **police cinématographique ou éditoriale**, avec :
     - Taille dominante (48–64px desktop, 28–32px mobile).
     - Espacement généreux (margin-top/bottom).
     - Style typographique immersif : serif ou sans-serif avec contraste, capitalisation soignée.
     - Option d’ombre portée ou contour léger pour donner du relief sans fond.
   - Le titre doit rester **lisible et impactant même si l’affiche est absente ou masquée**.

2. Bloc principal du média
   - Affiche (poster) placée à gauche ou en haut, mais **jamais fusionnée avec le titre**.
   - Score/notation visible (moyenne + nombre de votes, avec étoiles ou jauge circulaire).
   - Actions utilisateur :
     - Ajouter aux favoris.
     - Ajouter à la watchlist.
     - Noter (1–10).
     - Partager.
   - État non connecté → modal “Login Required!” avec options Close / Proceed to Login, redirection après connexion.

3. Métadonnées
   - Date de sortie (jour, mois, année, pays).
   - Durée formatée (ex. 2h 25m).
   - Genres cliquables (boutons ou tags).
   - Sociétés de production (logo + nom cliquable).
   - Réalisateurs, créateurs, showrunners (portrait + nom + rôle).
   - Pays d’origine, langue originale, budget, box-office, statut (Released, Upcoming).

4. Sections de contenu
   - Overview : synopsis complet, paragraphe lisible avec interligne suffisant.
   - Galerie d’images/backdrops : mosaïque responsive avec lazy loading, zoom au clic.
   - Collection : liste des autres titres liés avec affiche + année, navigation fluide.
   - Reviews :
     - Auteur + avatar.
     - Note (ex. Rated: 8/9/7).
     - Date relative (ex. 3 days ago).
     - Extrait du texte + bouton “Lire plus”.
   - Cast & Credits :
     - Grille de portraits (taille uniforme).
     - Nom + rôle (ex. Judy Hopps (voice)).
     - Bouton “Show All” pour charger plus.
   - Recommendations & Similar :
     - Grille de suggestions avec affiche + titre.
     - Hover → affiche résumé rapide.

5. Disposition et hiérarchie visuelle
   - Titre → Score → Actions → Métadonnées → Sections.
   - Grilles responsive : 2–6 colonnes selon écran.
   - Espacements réguliers entre blocs pour respirer.
   - Sections bien délimitées avec titres typographiques (H2/H3).

6. Interactions et comportements
   - Favoris/Watchlist persistants en base, visibles dans profil/historique.
   - Notation enregistrée et affichée dans le score global.
   - Genres/Production cliquables → recherche filtrée.
   - Modal login déclenché si action protégée.
   - Lazy loading pour images et cast.
   - Responsive : grille fluide, sections empilées sur mobile.
   - Infinite scroll ou pagination pour longues listes.
   - Notifications discrètes (toast) pour succès/erreur.

7. Accessibilité et qualité
   - Navigation clavier complète (tab order logique).
   - ARIA labels pour boutons, modals, images.
   - Texte lisible avec interligne suffisant et largeur max.
   - Contraste respectant WCAG 2.1 AA.
   - Feedback visuel pour focus et hover.
   - Modals accessibles (focus piégé, fermeture par ESC).

8. Performances
   - Minification CSS/JS.
   - Cache navigateur pour images et scripts.
   - CDN pour assets.
   - Lazy load pour médias lourds.
   - Préchargement des données critiques (titre, affiche, score).
   - SEO optimisé : balises meta, structured data (schema.org Movie/TVSeries).

9. Tests et critères d’acceptation
   - Titre toujours visible et lisible, même sans affiche.
   - Affichage complet : overview, collection, reviews, cast, recommendations.
   - Actions persistantes : favoris/watchlist/notation restent après refresh.
   - Modal login : apparaît correctement et redirige après connexion.
   - Responsive : page lisible sur desktop, tablette, mobile.
   - Performance : chargement < 3s, interactions < 250ms.
   - Accessibilité : navigation clavier, ARIA labels, contrastes validés.
   - Données cohérentes : genres, production, cast reliés à API.
